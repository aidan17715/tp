# Project Portfolio Page - Aidan (aidan17715)

## Product Overview
**SudoCook** is a command-line pantry and recipe management application designed for busy home cooks who want to reduce food waste and stay organized. Users can manage their ingredient inventory with expiry tracking, store recipes, and retrieve ingredients that are about to expire or have been stored the longest.

## Enhancements Implemented

### 1. Add Ingredient Command (`add-i`)
- **Purpose:** Allow users to add ingredients to their inventory with optional expiry dates
- **Command Format:** `add-i n/NAME q/QUANTITY u/UNIT [ex/YYYY-MM-DD]`
- **Example:** `add-i n/Milk q/2 u/liters ex/2026-04-10`
- **Files Modified/Created:**
  - `AddIngredientCommand.java` (new)
  - `Parser.java` (modified)
  - `Inventory.java` (modified)
  - `Ingredient.java` (modified)

### 2. Ingredient Expiry Dates
- **Purpose:** Track expiration dates to help users reduce food waste and identify ingredients that need to be used soon
- **Features:**
  - Expiry dates stored in YYYY-MM-DD format using Java's LocalDate
  - Integration with sort-i command for age-based sorting
  - Integration with list-i command for expiry filtering
  - Null-safe operations for ingredients without expiry dates
- **Files Modified:
  - `Ingredient.java` (extended with LocalDate field)
  - `Storage.java` (JSON serialization for expiry)
  - `ListIngredientCommand.java` (expiry filtering)
  - `SortInventoryCommand.java` (expiry-based sorting)

### 3. Data Persistence
- **Purpose:** Automatically save and load user data to ensure no loss of work
- **Implementation:**
  - JSON-based storage for recipes and ingredients
  - Automatic save on exit; automatic load on startup
  - Error handling for missing or corrupted files
- **Files Modified/Created:**
  - `Storage.java` (new)
  - `recipes.json` (auto-generated)
  - `inventory.json` (auto-generated)
  - `SudoCook.java` (modified to call Storage)

### 4. Undo Command
- **Purpose:** Allow users to revert recent modifications to inventory or recipes
- **Features:**
  - Supports up to 50 state snapshots
  - Single-command undo syntax: `undo`
  - Restores both Inventory and RecipeBook states
  - Deep copy snapshots for state isolation
- **Files Modified/Created:**
  - `UndoCommand.java` (new)
  - `CommandHistory.java` (new, manages snapshots)
  - `Parser.java` (modified)
  - `SudoCook.java` (modified to capture pre-command state)
